📑 Feature Spec: Nexus Vision System (NVS)1. ObjectiveEstablish a centralized, multi-consumer camera service on CORE-01 that provides low-latency live streaming, high-resolution AI snapshots, and automated recording for content creation.2. System Architecture: The "Producer-Consumer" ModelInstead of the CLI talking directly to the hardware, we introduce a Vision Daemon.The Producer: A background service that keeps the Logitech C920 open 24/7.The Shared Buffer: A memory-mapped file or a local Unix Socket where the latest frame is always available.The Consumers: Multiple modules (CLI, Recorder, Streamer) that pull from that buffer simultaneously.3. Functional ModulesA. The "Eye" (AI Snapshot Engine)Input: CLI Command (e.g., qraft "Nexus, inspect the print head").Technical Logic: The CLI requests the current frame from the buffer. It saves it as a high-quality .jpg.Integration: Injects the image into the Gemini 3.1 Flash multimodal prompt.Metadata: Appends Cerebro project context and current lab telemetry (temp/humidity) to the AI's visual reasoning.B. The "Chronicler" (Automated Recording)Function: Capture high-bitrate footage for the YouTube channel.Trigger: Manual (qraft rec start) or Event-based (e.g., Bambu Lab starts a print).Technical Logic: Uses ffmpeg to pull the raw MJPEG stream and transcode it to H.264/H.265 using the Vostro’s Intel QuickSync (VA-API) hardware acceleration to ensure 0% CPU impact.Storage: Saves to ~/qraftworx/vault/raw/.C. The "Viewport" (Live Streamer)Function: A stable, multi-client MJPEG or HLS stream for the MacBook Air HUD and remote viewing.Technical Logic: A FastAPI or Go web server that acts as a relay. It accepts the camera feed and "broadcasts" it.Stability: If a browser disconnects, the server stays alive. The "Pipe" never breaks.4. Technical Requirements & ImplementationI. InfrastructureService Manager: systemd (to ensure the camera service auto-restarts on boot).Hardware Acceleration: Use /dev/dri/renderD128 (Intel i7 iGPU) for all transcoding tasks.II. Remote Access ("Watch from Anywhere")Requirement: Access the feed without port-forwarding (which is insecure).Solution: Tailscale or WireGuard.Implementation: Install Tailscale on the Vostro and the Pi. This gives you a private, encrypted "Global Lab Network." You can view the stream on your phone via http://core.qraft:8080 even if you're at the hardware store.III. The Code StackLanguage: Python (via uv) or Go (if we bake it into Cerebro).Recommended Python Library: FastAPI + uvicorn. It handles asynchronous connections perfectly, preventing the "one-frame-freeze" issue.5.
